Ontology Review

Updated 25/3/2014 - list of upper ontologies with comments/ratings for possible use in AI applications.

Ontology Datasets

WordNet
http://wordnet.princeton.edu/wordnet/download/
qualifies as an upper ontology by including the most general concepts as well as more specialized concepts, related to each other not only by the subsumption relations, but by other semantic relations as well, such as part-of and cause. However, unlike Cyc, it has not been formally axiomatized so as to make the logical relations between the concepts precise. It has been widely used in Natural language processing research

SUMO - Suggested Upper Merged Ontology
http://www.ontologyportal.org/
includes an upper ontology, created by the IEEE working group P1600.1

DOLCE - Descriptive Ontology for Linguistic and Cognitive Engineering
http://www.loa-cnr.it/ontologies/DOLCE-Lite.owl
has a clear cognitive bias, in that it aims at capturing the ontological categories underlying natural language and human common sense

DBPedia
http://wiki.dbpedia.org/Datasets
The most comprehensive set of data based on Wikipedia (470M facts)

BFO - Basic Formal Ontology
http://www.ifomis.org/bfo
Incorporates both three-dimensionalist and four-dimensionalist perspectives on reality within a single framework. Has over 100 other ontologies build based on this

UMBEL
http://structureddynamics.com/resources.html#Ontologies
Maps to a simplified subset of the OpenCyc ontology (28,000 entries)

DnS - Descriptions and Situations
http://stlab.istc.cnr.it/stlab/The_Semantic_Technology_Laboratory_%28STLab%29
constructivist ontology that pushes DOLCEs descriptive stance even further allowing for context-sensitive redescriptions of the types and relations postulated by other given ontologies

GFO - General Formal Ontology
http://www.onto-med.de/
Unrated

UFO - Unified Foundation Ontology
http://
new, pretty good. tested for complex domains, combines DOLCE and GFO

CIDOC Conceptual Reference Model
http://en.wikipedia.org/wiki/CIDOC_Conceptual_Reference_Model
domain ontology, but includes its own version of an upper ontology in its core classes

COSMO - COmmon Semantic MOdel
http://404
The current (May 2009) OWL version of COSMO has over 6400 types (OWL classes), over 700 relations, and over 1400 restrictions

OCHRE
http://en.wikipedia.org/wiki/Object-centered_high-level_reference_ontology
has a focus on conceptual simplicity, so that the number of basic (primitive) concepts and relations is as small as possible in order to simplify the theory

YAMATO - Yet Another More Advanced Top Ontology
http://www.ei.sanken.osaka-u.ac.jp/hozo/onto_library/upperOnto.htm
complex but very advanced

KSL Omtonlingua
http://www.ksl.stanford.edu/software/ontolingua/
Unrated

DAML Ontology Library
http://www.daml.org/ontologies/
Unrated

OpenCyc
http://en.wikipedia.org/wiki/Cyc#ResearchCyc
Proprietry but fairly precise

PROTON
http://
basic subsumption hierarchy which provides coverage of most of the upper-level concepts necessary for semantic annotation, indexing, and retrieval

IDEAS
http://
not intended for reasoning and inference purposes; its purpose is to be a precise model of business

MarineTLO
http://www.ics.forth.gr/isl/MarineTLO/
Low - aimed for Marine domain

DIY - eg build your own Ontology
http://localhost
Not ideal - better to use existing sets, but best method to get a concise set of tuples



Useful Links for Ontological development

A Comparison of Upper Ontologies (Technical Report DISI-TR-06-21)
http://www.disi.unige.it/person/MascardiV/Download/DISI-TR-06-21.pdf
Summary of ontologies - compares BFO, cyc, DOLCE, GFO, PROTON, Sowas, SUMO

Ontology-based data integration
http://en.wikipedia.org/wiki/Ontology-based_data_integration
short article with examples of approaches to use

Python RDF library
https://github.com/RDFLib/rdflib
good simple examples for using RDFLIB at https://rdflib.readthedocs.org/en/latest/

Protege - ontology editor
http://protege.stanford.edu/products.php#desktop-protege
feature rich ontology editing environment with full support for the OWL 2 Web Ontology Language, and direct in-memory connections to description logic reasoners like HermiT and Pellet

ontogenesis - upper level ontologies
http://ontogenesis.knowledgeblog.org/740
basic overview of ontologies with descriptions of time/space and objects/processes

Toward the Use of an Upper Ontology for U.S. Government and U.S. Military Domains: An Evaluation
http://oai.dtic.mil/oai/oai?verb=getRecord&metadataPrefix=html&identifier=ADA459575
very good explanation of the types of ontological choices such as 3d/4d, descriptive vs revisionary, mult vs reduct, ...

ROMULUS - A repository of foundational ontologies
http://www.thezfiles.co.za/ROMULUS/downloads.html
Very good list of links to ontologies

Ontological realism: A methodology for coordinated evolution of scientic ontologies
http://iospress.metapress.com/content/1551884412214u67/fulltext.pdf
technical focus on biological and biomedical ontologies within the framework of the OBO (Open Biomedical Ontologies) Foundry initiative

Ontology Development 101: A Guide to Creating Your First Ontology
http://protege.stanford.edu/publications/ontology_development/ontology101-noy-mcguinness.html
Good introduction and examples of building an ontology - key points: reuse parts if possible, but build yourself to keep it short and valid

Onto-Med Report Nr. 8
http://www.onto-med.de/publications/2010/gfo-basic-principles.pdf
basic principles of GFO
